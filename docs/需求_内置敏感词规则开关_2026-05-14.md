# 需求：内置敏感词扫描规则开关

**创建时间**：2026-05-14  
**版本**：v1.0  
**状态**：待实施  
**优先级**：P1（重要）

---

## 📋 需求背景

当前 DataGuard Scanner 默认启用所有内置敏感词扫描规则（身份证号、手机号、邮箱、银行卡号、地址、IP地址、密码等），用户无法选择仅使用自定义关键字搜索功能。

**内置规则包括**：
- 🆔 身份证号检测（18位中国居民身份证）
- 📱 手机号检测（中国大陆11位手机号）
- 📧 邮箱检测（标准邮箱格式）
- 💳 银行卡号检测（借记卡/信用卡）
- 🏠 地址检测（中国行政区划地址）
- 🌐 IPv4地址检测（IP地址格式）
- 🔑 密码检测（password/pwd等关键词）
- 👤 中文姓名检测（2-4个连续汉字，可选）

部分用户场景下，只需要进行简单的关键字匹配，不需要复杂的敏感信息检测，希望禁用内置规则以提升扫描速度和简化结果展示。

---

## 🎯 需求目标

1. **提供灵活的扫描模式选择**：用户可以选择启用/禁用内置敏感词规则
2. **优化纯关键字扫描体验**：禁用内置规则后，界面更简洁，专注于关键字匹配
3. **保持功能完整性**：两种模式互不干扰，切换时状态正确保存和恢复

---

## ✨ 功能需求

### 1. 设置项：启用内置敏感词扫描规则

#### 1.1 位置
- **路径**：设置窗口 → 扫描配置区域
- **控件类型**：复选框（Checkbox）
- **标签文本**：`启用内置敏感词扫描规则`
- **默认值**：✅ 勾选（启用）

#### 1.2 说明文字
```
启用后将检测以下敏感信息：
• 身份证号、手机号、邮箱、银行卡号
• 地址、IP地址、密码、中文姓名

取消勾选则仅使用自定义表达式进行关键字搜索。
```

#### 1.3 交互逻辑
- ✅ **勾选时**：
  - `enabledSensitiveTypes` 配置生效
  - 显示敏感类型选择列表（身份证、手机号等）
  - 结果表格显示"敏感信息数量"列
  - 结果表格显示操作列的删除按钮
  - 显示多选时的一键删除按钮
  - 开始扫描按钮始终可用（即使没有自定义表达式）

- ❌ **取消勾选时**：
  - 忽略 `enabledSensitiveTypes` 配置
  - 隐藏敏感类型选择列表
  - 结果表格隐藏所有检测列（敏感类型、Total、表达式）
  - 结果表格隐藏操作列的删除按钮
  - 隐藏多选时的一键删除按钮
  - **自定义表达式必须输入且语法正确，否则开始扫描按钮禁用**
  - 仅使用自定义表达式进行关键字搜索

#### 1.4 配置持久化
- **存储位置**：用户配置文件（config.json）
- **字段名**：`enableBuiltinRules: boolean`
- **默认值**：`true`

### 2. 结果表格动态调整

#### 2.1 隐藏所有检测结果列

**触发条件**：
- 当 `enableBuiltinRules === false` 时，隐藏以下所有列：
  1. **敏感信息类型列**（身份证、手机号、邮箱等）
  2. **敏感信息总数列**（Total）
  3. **自定义表达式列**（如果配置了表达式）

**原因说明**：
- 禁用内置规则后，`counts` 对象为空 `{}`，没有任何敏感类型计数
- **自定义表达式仍然有效**：用户手动输入的表达式用于关键字搜索，与内置规则无关
- 但为保持界面简洁和语义一致，关闭内置规则时隐藏所有检测相关列（包括表达式列）
- 用户此时仅关注"哪些文件包含关键字"，不关心具体的敏感信息统计

**实现方式**：
```typescript
// ResultsTable.vue
const showDetectionColumns = computed(() => {
  return config.value.enableBuiltinRules !== false
})

// 敏感类型列
const sensitiveTypes = computed(() => {
  if (!showDetectionColumns.value) {
    return []  // 返回空数组，不渲染任何敏感类型列
  }
  return allSensitiveTypes.value.filter(type => 
    config.value.enabledSensitiveTypes.includes(type.id)
  )
})

// 表达式列
const hasCustomExpressionColumn = computed(() => {
  // 只有在启用内置规则时，才显示表达式列
  if (!showDetectionColumns.value) {
    return false
  }
  return scanResults.value.some(item =>
    item.expressionMatched !== undefined && item.expressionMatched !== null
  )
})
```

**影响范围**：
- **表头**：不渲染敏感类型列、Total 列、表达式列
- **数据行**：不渲染对应的计数单元格和对勾图标
- **Grid 模板**：动态调整列数，只保留基础列（复选框、文件名、大小、时间、操作）
- **固定列宽度计算**：`fixedColumnsTotalPx` 自动排除检测列的宽度（已有逻辑）
- **路径列宽度**：**保持原有逻辑不变**（ResizeObserver + `minmax(8em, 1fr)`），无需修改

**说明**：
- 原有的 `gridStyle` 计算属性中，path 列使用 `minmax(8em, 1fr)`
- `fixedColumnsTotalPx` 会根据 `sensitiveTypes.value.length` 和 `hasCustomExpressionColumn.value` 自动计算
- 当禁用内置规则时，`sensitiveTypes` 为空数组，`hasCustomExpressionColumn` 为 false
- `fixedColumnsTotalPx` 自动减小，路径列自动获得更多空间
- **无需额外代码**，原有逻辑已完美支持

### 3. 扫描逻辑调整

#### 3.1 后端 Worker 处理

**当 `enableBuiltinRules === false` 时，禁用的规则包括**：
- ❌ **身份证号检测**：18位中国居民身份证正则匹配 + 校验码验证
- ❌ **手机号检测**：中国大陆11位手机号正则匹配 + 号段验证
- ❌ **邮箱检测**：标准邮箱格式正则匹配
- ❌ **银行卡号检测**：借记卡/信用卡正则匹配 + Luhn算法校验
- ❌ **地址检测**：中国行政区划地址正则匹配
- ❌ **IPv4地址检测**：IP地址格式正则匹配 + 范围验证
- ❌ **密码检测**：password/pwd等关键词模式匹配
- ❌ **中文姓名检测**：2-4个连续汉字正则匹配（如果启用）

**不禁用的功能**：
- ✅ **自定义表达式搜索**：用户手动输入的逻辑表达式（如：`密码 & 身份证`）
- ✅ **文件解析**：PDF、Excel、Word 等文件格式的文本提取
- ✅ **流式处理**：大文件的分块读取和处理
- ✅ **结果过滤**：根据表达式匹配结果过滤文件列表

**执行逻辑**：
```typescript
// file-worker.ts
const { 
  enableBuiltinRules = true,  // 【新增】默认启用
  enabledSensitiveTypes,
  customExpression,
  // ...
} = task;

let counts: Record<string, number> = {};
let totalCount = 0;

if (enableBuiltinRules) {
  // ✅ 执行所有内置规则检测
  const result = detectSensitiveData(text, enabledSensitiveTypes);
  counts = result.counts;      // { idCard: 2, phone: 5, email: 3, ... }
  totalCount = result.total;   // 10
} else {
  // ❌ 跳过所有内置规则检测
  counts = {};                 // 空对象
  totalCount = 0;              // 总数为 0
}

// ✅ 始终执行自定义表达式评估（无论 enableBuiltinRules 的值）
if (customExpression && customExpression.trim()) {
  const isMatched = evaluateCustomExpression(text, customExpression);
  if (isMatched) {
    result.expressionMatched = 1;
  }
}

// 返回结果
parentPort?.postMessage({
  taskId,
  filePath,
  counts,                      // 禁用时为 {}
  total: totalCount,           // 禁用时为 0
  expressionMatched: result.expressionMatched || 0,  // 可能为 0 或 1
  // ...
});
```

### 4. UI 状态同步

#### 4.1 工具栏开始扫描按钮验证

**当 `enableBuiltinRules === false` 时**：
- ✅ **必须输入自定义表达式**，且语法正确，才能启用“开始扫描”按钮
- ❌ 如果表达式为空或语法错误，按钮保持禁用状态

**实现方式**：
```vue
<!-- App.vue - 工具栏 -->
<button
  class="btn btn-primary btn-scan"
  :disabled="!canStartScan"
  @click="handleStartScan"
>
  <svg><use href="#icon-scan"/></svg>
  <span>开始扫描</span>
</button>
```

```typescript
// App.vue - 计算属性
const canStartScan = computed(() => {
  // 如果启用了内置规则，始终可以扫描
  if (config.value.enableBuiltinRules !== false) {
    return !isScanning.value && selectedPaths.value.length > 0
  }
  
  // 如果禁用了内置规则，必须有有效的自定义表达式
  if (!customExpression.value.trim()) {
    return false  // 表达式为空
  }
  
  // 检查表达式语法是否正确
  if (expressionError.value) {
    return false  // 表达式有语法错误
  }
  
  return !isScanning.value && selectedPaths.value.length > 0
})
```

**用户体验**：
- 表达式输入框实时验证（500ms 防抖）
- 语法错误时，输入框边框变红，显示错误提示
- 开始扫描按钮根据验证结果动态启用/禁用
- 鼠标悬停在禁用按钮上时，显示提示：“请输入有效的自定义表达式”
