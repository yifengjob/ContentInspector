# 自定义敏感词逻辑搜索功能需求文档

**创建日期**: 2026-05-12  
**版本**: v1.0

## 🎯 核心需求

在顶部工具栏右侧添加一个输入框，支持用户输入自定义敏感词，并使用**逻辑表达式**进行匹配。

---

## 📝 详细需求说明

### 1. UI 需求

**位置**：顶部工具栏右侧  
**组件**：文本输入框  
**占位符提示**：例如 "输入敏感词（支持 !、()、&、|）"  
**样式**：与其他工具栏元素保持一致

---

### 2. 功能需求

#### 2.1 基本功能

- 用户可以在输入框中输入自定义敏感词
- 扫描时，将用户输入的文本作为额外的敏感词规则进行匹配
- 匹配结果与系统内置的敏感词规则合并显示

#### 2.2 逻辑表达式支持

支持以下逻辑运算符：

| 运算符 | 含义 | 示例 | 说明 |
|--------|------|------|------|
| `!` | 非（NOT） | `!密码` | 不包含"密码" |
| `&` | 与（AND） | `密码 & 身份证` | 同时包含"密码"和"身份证" |
| `\|` | 或（OR） | `密码 \| 身份证` | 包含"密码"或"身份证" |
| `()` | 分组 | `(密码 \| 身份证) & 机密` | 先计算括号内，再与"机密"做与运算 |

#### 2.3 优先级规则

**优先级从高到低**：
1. `()` - 括号分组（最高优先级）
2. `!` - 非运算
3. `&` - 与运算
4. `\|` - 或运算（最低优先级）

**示例**：
```
!密码 & (身份证 | 银行卡)
```

**解析顺序**：
1. 先计算 `(身份证 | 银行卡)` → 结果为 A
2. 再计算 `!密码` → 结果为 B
3. 最后计算 `B & A` → 最终结果

---

### 3. 技术实现要点

#### 3.1 表达式解析器

需要实现一个**逻辑表达式解析器**，包括：

##### 3.1.1 词法分析（Tokenizer）
- 将输入字符串分解为 token（关键词、运算符、括号）
- 识别：普通文本、`!`、`&`、`\|`、`(`、`)`

##### 3.1.2 语法分析（Parser）
- 将 token 序列转换为**抽象语法树（AST）**
- 遵循优先级规则构建正确的树结构

##### 3.1.3 求值器（Evaluator）
- 遍历 AST，对每个节点进行匹配判断
- 返回最终的布尔结果（是否匹配）

#### 3.2 AST 节点类型

```typescript
type ASTNode = 
  | { type: 'keyword'; value: string }          // 关键词节点
  | { type: 'not'; child: ASTNode }              // 非运算节点
  | { type: 'and'; left: ASTNode; right: ASTNode }  // 与运算节点
  | { type: 'or'; left: ASTNode; right: ASTNode };  // 或运算节点
```

#### 3.3 匹配逻辑

对于文件内容中的每一行（或整个文本）：

```typescript
function evaluate(node: ASTNode, text: string): boolean {
  switch (node.type) {
    case 'keyword':
      return text.includes(node.value);
    
    case 'not':
      return !evaluate(node.child, text);
    
    case 'and':
      return evaluate(node.left, text) && evaluate(node.right, text);
    
    case 'or':
      return evaluate(node.left, text) || evaluate(node.right, text);
  }
}
```

### 4. 集成点

#### 4.1 配置管理

- 在 `ScanState` 或配置中添加 `customSensitiveExpression: string` 字段
- 支持保存和加载用户的自定义表达式

#### 4.2 扫描流程

- 在敏感数据检测阶段（`sensitive-detector.ts`）
- 除了匹配内置规则外，还要评估自定义表达式
- 如果表达式匹配成功，标记为敏感文件

#### 4.3 IPC 通信

- 新增 IPC handler：`set-custom-expression` - 设置自定义表达式
- 新增 IPC handler：`get-custom-expression` - 获取当前表达式

---

### 5. 边界情况处理

#### 5.1 输入验证

- **空字符串**：忽略自定义规则
- **语法错误**：显示错误提示，不使用该规则
- **特殊字符转义**：支持转义 `!`、`&`、`\|`、`(`、`)` 等字符

#### 5.2 性能考虑

- **表达式缓存**：避免重复解析同一表达式
- **短路求值**：`&` 和 `\|` 运算支持短路优化
- **大文件优化**：流式处理时逐行评估，避免加载全部内容

#### 5.3 用户体验

- **实时语法检查**：输入时显示语法是否正确
- **错误提示**：明确指出语法错误位置
- **示例帮助**：提供常用表达式示例

### 6. 测试用例

#### 6.1 基础测试

```
输入: "密码"
文本: "这是密码测试"
结果: ✅ 匹配

输入: "!密码"
文本: "这是密码测试"
结果: ❌ 不匹配

输入: "!密码"
文本: "这是测试文档"
结果: ✅ 匹配
```

#### 6.2 组合测试

```
输入: "密码 & 身份证"
文本: "密码:123, 身份证:456"
结果: ✅ 匹配

输入: "密码 & 身份证"
文本: "只有密码"
结果: ❌ 不匹配

输入: "密码 | 身份证"
文本: "只有密码"
结果: ✅ 匹配
```

#### 6.3 优先级测试

```
输入: "!密码 & (身份证 | 银行卡)"
文本: "没有密码，有身份证"
结果: ✅ 匹配（!密码=true, 身份证=true, true & true = true）

输入: "!密码 & (身份证 | 银行卡)"
文本: "有密码，有身份证"
结果: ❌ 不匹配（!密码=false, false & true = false）
```

#### 6.4 边界测试

```
输入: ""
结果: 忽略规则

输入: "密码 &"
结果: 语法错误提示

输入: "(密码"
结果: 括号不匹配错误
```

⚠️ **关键点**：

1. **优先级必须正确**：`( ) > ! > & > |`
2. **错误处理要友好**：清晰的错误提示
