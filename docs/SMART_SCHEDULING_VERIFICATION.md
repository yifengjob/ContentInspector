# 智能调度验证指南

## 📋 验证目标

确认智能调度完全按照您的要求工作：

> **核心原则**：当队列中找不到符合类型相斥规则的文件时，应该立即从队列中选一个不违反大文件规则的任务直接执行，不能让worker闲置，只有一种情况能让worker存在闲置的情况，那就是大文件规则。

---

## 🔍 验证步骤

### **1. 准备测试环境**

```bash
# 确保在开发模式运行（才能看到详细日志）
npm run dev
```

### **2. 选择测试目录**

建议选择一个包含多种文件类型的目录，例如：
- `/Users/yourname/Documents`（包含 docx, pdf, xlsx, txt 等）
- 或者创建一个测试目录，包含以下文件：
  - `test1.xlsx` (5MB) - Excel 大文件
  - `test2.pdf` (15MB) - PDF 大文件
  - `test3.docx` (8MB) - Word 大文件
  - `test4.txt` (10KB) - 文本小文件
  - `test5.log` (5KB) - 日志小文件

### **3. 观察日志输出**

启动扫描后，在终端或日志文件中查找 `[智能调度]` 开头的日志。

## ✅ 验证点 1：策略 1 - 优先大文件

**预期日志：**
```
[智能调度] 状态: 队列=5, 处理中类型=[], 大文件=0/1
[智能调度] 策略1: 优先处理大文件 test2.pdf (15.0MB)
[智能调度] 分配任务 [0] test2.pdf (pdf, 15.0MB)
```

**验证标准：**
- ✅ 如果队列中有大文件且 `largeFilesProcessing < MAX_LARGE_FILES_CONCURRENT`，应该优先选择大文件
- ✅ 同时检查该大文件的类型是否被阻塞

---

## ✅ 验证点 2：策略 2 - 类型互斥

**预期日志：**
```
[智能调度] 状态: 队列=4, 处理中类型=[pdf:1], 大文件=1/1
[智能调度] 策略2: 选择不同类型文件 test1.xlsx (excel)
[智能调度] 分配任务 [1] test1.xlsx (excel, 5.0MB)
```

**验证标准：**
- ✅ Worker 0 正在处理 PDF
- ✅ Worker 1 应该选择非 PDF 类型的文件（如 Excel）
- ✅ 不应该出现两个 Worker 同时处理同类型文件

---

## ✅ 验证点 3：策略 3 - 无新类型时立即处理

**场景：** 4 个 Worker，正在处理 TXT/PDF/XLSX/WORD，队列中也只有这 4 种类型

**预期日志：**
```
[智能调度] 状态: 队列=4, 处理中类型=[txt:1, pdf:1, xlsx:1, word:1], 大文件=1/1
[智能调度] 策略3: 无新类型，优先选择小文件 test4.txt
[智能调度] 分配任务 [2] test4.txt (text, 0.0MB)
```

**验证标准：**
- ✅ 队列中的类型都已经在处理中（无新类型）
- ✅ Worker 不应该等待 5 秒超时
- ✅ 应该立即选择一个小文件处理

---

## ✅ 验证点 4：策略 5 - 兜底逻辑（关键！）

### **场景 A：所有类型被阻塞，但有小文件**

**预期日志：**
```
[智能调度] 状态: 队列=3, 处理中类型=[excel:1, pdf:1, word:1], 大文件=1/1
[智能调度] 策略5: 所有类型被阻塞，选择小文件 test4.txt
[智能调度] 分配任务 [2] test4.txt (text, 0.0MB)
```

**验证标准：**
- ✅ 即使违反类型互斥，也应该选择小文件
- ✅ Worker 不应该闲置

### **场景 B：所有类型被阻塞，只有大文件但未达上限**

**预期日志：**
```
[智能调度] 状态: 队列=2, 处理中类型=[excel:1, pdf:1], 大文件=0/1
[智能调度] 策略5: 所有类型被阻塞，大文件未达上限，选择 test5.xlsx
[智能调度] 分配任务 [2] test5.xlsx (excel, 12.0MB)
```

**验证标准：**
- ✅ 即使违反类型互斥，也应该选择大文件（因为未达上限）
- ✅ Worker 不应该闲置

### **场景 C：全是大文件且已达上限（唯一让 Worker 闲置的情况）**

**预期日志：**
```
[智能调度] 状态: 队列=2, 处理中类型=[excel:1, pdf:1], 大文件=1/1
[智能调度] 策略5: 全是大文件且已达上限，Worker 等待中...
```

**验证标准：**
- ✅ Worker 应该闲置等待
- ✅ 这是唯一允许 Worker 闲置的情况

## 🐛 常见问题排查

### **问题 1：看不到 `[智能调度]` 日志**

**原因：** 可能不在开发模式运行

**解决：**
```bash
# 确保使用 dev 命令
npm run dev
```

### **问题 2：同类型文件并发处理**

**原因：** while 循环逻辑有问题

**检查：** 查看日志中是否有类似这样的记录：
```
[智能调度] 分配任务 [0] file1.xlsx (excel)
[智能调度] 分配任务 [1] file2.xlsx (excel)  ← ❌ 错误！
```

应该是：
```
[智能调度] 分配任务 [0] file1.xlsx (excel)
[智能调度] 状态: 处理中类型=[excel:1]
[智能调度] 策略2: 选择不同类型 file2.pdf (pdf)  ← ✅ 正确！
```

### **问题 3：Worker 闲置超过 5 秒**

**原因：** 策略 3 或策略 5 没有正确触发

**检查：** 查看是否有这样的日志：
```
[智能调度] 策略5: 全是大文件且已达上限，Worker 等待中...
```

如果没有，说明有其他问题。

## 🎯 最终验证标准

**完全符合要求的标志：**

1. ✅ **类型互斥优先遵守** - 大多数情况下，不同 Worker 处理不同类型
2. ✅ **Worker 不闲置** - 除非全是大文件且已达上限，否则 Worker 立即处理任务
3. ✅ **大文件限制严格遵守** - 最多只有 `MAX_LARGE_FILES_CONCURRENT` 个大文件并发
4. ✅ **状态同步正确** - 任务完成后，计数正确减少，下一个任务能立即调度
5. ✅ **while 循环生效** - 每次分配后立即重新检查状态，不会一次性分配多个同类型任务
